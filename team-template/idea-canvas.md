# 💡 Idea Canvas — Lean Canvas (4 ช่องแรก)

> ทำช่วงท้ายวัน เปลี่ยน "ของที่รันได้วันนี้" → "ไอเดียนวัตกรรมที่ต่อยอด Day 2 ได้"
> เริ่มจาก **ปัญหา** ไม่ใช่ "เรามี sensor อะไร" · ทำ 4 ช่องนี้ให้คมก่อน ที่เหลือไว้ทีหลัง

- **ชื่อ Prototype (ชั่วคราว):** FaintCam Detect

---

## 1️⃣ Problem — ปัญหาคืออะไร

เขียน 2–3 ข้อ ที่มีคน "ลำบาก / เสียเวลา / พลาด" จริง:
```
- ตรวจจับอาการล้มของมนุษย์เพื่อช่วยเหลือได้ทันท่วงทีจากการใช้ AI ตรวจจับ
- อาจเสียชีวิตหากได้รับการช่วยเหลือไม่ทันท่วงที
- อาจเสียทั้งชีวิตและทรัพสินหากพลาดเพียงครั้งเดียว
```
**ตอนนี้คนแก้ปัญหานี้ยังไง (ของเดิม)?** ได้รับความช่วยเหลือที่ไม่ทันท่วงที

## 2️⃣ Customer — ใครคือผู้ใช้จริง

เจาะให้ชัด ไม่ใช่ "ทุกคน":
```
ทุกคนและผู้ที่มีอาการป่วยหรือผู้สูงอายุที่อาจจะไม่สามารถช่วยได้ทันท่วงที
```

## 3️⃣ Value (USP) — คุณค่าเด่นในประโยคเดียว

```
"[FaintCam Detect] ช่วย [ผู้คน] [ที่สลบได้ทันท่วงที]
 ต่างจาก [hand and face detection] เพราะ [สามารถตรวจจับคนและท่าทางมนุษยได้ซึ่งมีประโยชน์กว่า]" โดยจำเป็นกล้องที่ตั้งในที่สาธารณะโดยสามารถตรวจจับโดยผู้ใช้ไม่จำเป็นต้องใส่อุปกรณ์เสริมใดๆ
```
ตัวอย่าง: *"SafeFall ช่วยผู้สูงอายุที่อยู่คนเดียว ขอความช่วยเหลือได้ทันเมื่อล้ม — ต่างจาก smartwatch แพงๆ เพราะใช้อุปกรณ์ราคาถึงได้และไม่ต้องต่อเน็ต"*

## 4️⃣ Solution — แก้ด้วยอะไร

- **Input ที่ใช้:** กล้อง(จับท่ทาง) (Modulino ตัวไหน / กล้อง / ไมค์)
- **Class ที่ AI แยก:** Stand,Faith
- **Output ตอบสนอง:** Buzzer (Pixels / Buzzer / จอ / log)

---

## 🎯 MVP — ฟังก์ชันเดียวที่ต้องสาธิตให้ได้

```
ตรวจจับท่าทางและแยกได้
```

**1 core function ที่จะโชว์ Day 2/3:** ตรวจจับว่า Stand/Faith แล้วมีเสียงเตือนและไฟ
*(เช่น "ตรวจได้ว่า Safe/Warning แล้วไฟ+เสียงเตือน")*

> คาถา: **Build less, prove more** การ detect โดยที่ตรวจจับ ท่าทางสรีระแล้วมี sensor g9nvo
